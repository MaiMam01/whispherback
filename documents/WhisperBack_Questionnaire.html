These first two sections cover the basics — which phones the app should work on, and how it should behave when real life happens (calls, other apps, the screen turning off).
01 Devices & Platforms

Which devices the app should work on.

1.1
Should the app launch on Android, iPhone, or both? If both, which one would you like us to focus on first?
1.2
Would you prefer one shared app for both Android and iPhone (faster, lower cost) or two separate apps tailored to each (smoother, slightly more expensive)?
1.3
How old can users’ phones be? For example, only phones from the last 4–5 years, or do we need to support older models too?
1.4
Should the app work nicely on tablets (iPad, Android tablets) at launch too, or just phones for now?
1.5
Will users want to use WhisperBack on a computer through a website as well, or is the phone app enough for launch?
02 How Playback Should Behave

How clips should play when life gets in the way.

2.1
When the user sets a gap between clips (say 10 minutes), should we count that gap from when one clip ends, or from when one clip starts to when the next starts?
2.2
If a clip is meant to play during a phone call or while the user is listening to music in another app, should we skip it, save it for later, or wait quietly until things are free?
2.3
On Android, to make sure clips play exactly on time, the app needs to show a small notification while it’s active (similar to Spotify or YouTube Music). Are you OK with that?
2.4
On iPhones, Apple sometimes delays apps in the background by 1–2 minutes to save battery. Is that small delay acceptable, or does timing need to be exact?
2.5
If a scheduled clip is about to play but the user is already playing something manually in the app, what should happen — schedule waits, manual track stops, or both stack?
2.6
If a user tries to delete a playlist that has an active schedule on it, should we just cancel the schedule and delete it, or stop them and ask “are you sure?”
2.7
Should the app automatically play through Bluetooth headphones, AirPods, and car speakers, or stick to the phone speaker unless the user changes it?
2.8
If the phone is on silent or Do-Not-Disturb, should scheduled clips stay quiet too, or play anyway because the user set them up on purpose?

03 Recording & Audio

Recording, importing, and editing — the audio basics.

3.1
What recording quality should we aim for — standard (clear voice, smaller files) or higher (richer sound, larger files)?
3.2
What’s a sensible maximum length for a single clip — for example 5 minutes? And a maximum file size, like 10 MB?
3.3
Which audio file types should users be able to import — common ones like MP3 only, or also more specialised formats?
3.4
Should users be able to do basic editing inside the app — like trimming a clip, adding fade in/out, or evening out the volume?
3.5
When users record a clip, should we automatically clean up background noise and balance the volume, or keep their recording exactly as-is?
3.6
When a user imports an audio file from their phone, should the app keep its own safe copy (always works, uses more space) or just point to the original (saves space, but breaks if the file is moved)?
04 Accounts, Cloud & Pricing

Sign-in, cloud backup, and how the app makes money.

4.1
Any preference on which cloud provider hosts the app’s data — Google, Amazon, or Microsoft — or are you happy for us to recommend?
4.2
Is there a region where user data must legally be stored — for example Europe, the Gulf, or the US — or is any region fine?
4.3
Should the app back up playlists to the cloud automatically as the user makes changes, or only when they tap a “Backup” button themselves?
4.4
Should a user be able to use the same account on more than one phone at the same time (we’ll handle conflicts), or only one device at a time?
4.5
How much cloud storage should each Premium user get — say 1 GB, 5 GB, or unlimited? More storage means higher running costs.
4.6
Which sign-in options should we offer — Google only, or also Apple, email + password, and phone-number with one-time code?
4.7
Should users have to create an account to use the app, or can they use the basic features without signing in (and only sign in if they want Premium)?
4.8
How should the app make money — fully free, a one-time payment to unlock everything, or a monthly / yearly subscription? And do you want a free trial?
4.9
Apart from cloud backup, what other features should be locked behind Premium — bigger playlist limits, web access, higher recording quality, anything else?

05 Look, Feel & Languages

Languages, theme, and accessibility for everyone.

5.1
Which languages should the app support at launch — English only, or also Urdu, Arabic, Hindi, or others?
5.2
Arabic and Urdu are read right-to-left. Should the entire layout flip for those languages (proper local feel), or should only the text be translated while the layout stays left-to-right?
5.3
Should the app open in dark mode, light mode, or quietly match whatever theme the user has set on their phone?
5.4
The app uses mostly icons. Should we add support for screen readers (so visually impaired users can use it) and an option to show text labels for everyone else?
06 Notifications & Privacy

Alerts, data handling, and post-launch quality.

6.1
Which moments should send a notification — a schedule starting, sleep mode beginning or ending soon, an import finishing, or something going wrong? Should the user be able to switch each one on or off?
6.2
Are you OK with us using standard tools to see how the app is being used and to catch crashes early? If you have a preferred tool, please share — otherwise we’ll pick a good one.
6.3
Should the user’s clips and playlists be extra-secured (encrypted) on their phone, on top of the standard protection that Android and iPhone already provide?
6.4
When a user deletes their account, should we wipe their cloud data right away, or hold it for a few days in case they change their mind?
6.5
Will you provide the privacy policy and terms of service text, or would you like us to draft a starter version for your review?
6.6
Are there any age restrictions for the app — for example only 13 and above — or no age limits at all?
07 Launch & Scope

Timeline, store accounts, and what we’re proposing for v1.

7.1
When are you hoping to launch? And are there any specific dates we should plan around — beta release, store submission, marketing campaigns?
7.2
Do you already have an Apple Developer account and a Google Play publisher account, or should we set them up for you?
7.3
For v1.0 we’re proposing: Active/Inactive switch, Playlists, manual playback, Scheduling, Shuffle, and basic Sleep Mode — all stored on the phone only. Happy with that, or should we include cloud backup in v1 as well?
7.4
Would you like a small private testing round (where a hand-picked group tries the app on their phones) before the public launch?
7.5
Anything else on your mind that isn’t in the current document but should be planned for?
Next Step

Once we have your answers, we’ll send back WhisperBack Spec v1.1 — with the final feature list, screen designs, a simple flow showing how playback works, and a fixed-price proposal with timeline. Expected turnaround on our side: 5 working days from your reply.
